Для начала нужно создать Suite (Набор тестов). Это будет неким контейнером для тестов. 
Дальше создадим тестовый кейс для группы операций(по 1 признаку на кейс) 
Иерархически получится так 
📁 Suite (Набор) — верхний уровень организации
   └── 📂 TCase (Группа тестов) — средний уровень
        └── 📄 Test (Тест) — отдельная проверка

Suite — это коробка. TCase — это разделители внутри коробки. Тесты — это содержимое

1. функция `tcase_add_test` для добавления Теста в группу тестов TCase
2. функция `suite_add_tcase` для добавления группы тестов Tcase в набор Suite 

Конкретный пример: 
```C
#include <check.h>

// --- Допустим, у нас есть калькулятор с разными операциями ---

// Тесты для сложения (одна логическая группа)
START_TEST(test_add_positive) { ck_assert_int_eq(add(2,2), 4); } END_TEST
START_TEST(test_add_negative) { ck_assert_int_eq(add(-1,1), 0); } END_TEST
START_TEST(test_add_zero)     { ck_assert_int_eq(add(0,5), 5); } END_TEST

// Тесты для вычитания (другая логическая группа)
START_TEST(test_sub_positive) { ck_assert_int_eq(sub(5,3), 2); } END_TEST
START_TEST(test_sub_negative) { ck_assert_int_eq(sub(0,3), -3); } END_TEST
START_TEST(test_sub_zero)     { ck_assert_int_eq(sub(7,0), 7); } END_TEST

// Тесты для умножения (ещё одна группа)
START_TEST(test_mul_positive) { ck_assert_int_eq(mul(3,4), 12); } END_TEST
START_TEST(test_mul_zero)     { ck_assert_int_eq(mul(5,0), 0); } END_TEST

int main(void) {
    // 📁 Создаём ОДИН Suite для всего модуля "Калькулятор"
    Suite *s = suite_create("Calculator Module");
    
    // 📂 Создаём ПЕРВЫЙ TCase для операций сложения
    TCase *tc_add = tcase_create("Addition Tests");
    tcase_add_test(tc_add, test_add_positive);
    tcase_add_test(tc_add, test_add_negative);
    tcase_add_test(tc_add, test_add_zero);
    suite_add_tcase(s, tc_add);  // Добавляем TCase в Suite
    
    // 📂 Создаём ВТОРОЙ TCase для операций вычитания
    TCase *tc_sub = tcase_create("Subtraction Tests");
    tcase_add_test(tc_sub, test_sub_positive);
    tcase_add_test(tc_sub, test_sub_negative);
    tcase_add_test(tc_sub, test_sub_zero);
    suite_add_tcase(s, tc_sub);
    
    // 📂 Создаём ТРЕТИЙ TCase для операций умножения
    TCase *tc_mul = tcase_create("Multiplication Tests");
    tcase_add_test(tc_mul, test_mul_positive);
    tcase_add_test(tc_mul, test_mul_zero);
    suite_add_tcase(s, tc_mul);
    
    // Запускаем всё
    SRunner *sr = srunner_create(s);
    srunner_run_all(sr, CK_NORMAL);
    
    return srunner_ntests_failed(sr) == 0 ? 0 : 1;
}
```


Check предлагает современные и информативные макросы с префиксом `ck_`

- `ck_assert_int_eq(x, y)`: Проверяет, равны ли `x` и `y` (для `int`). При ошибке выводит значения.
- `ck_assert_int_ne(x, y)`: Проверяет неравенство.
- `ck_assert_str_eq(x, y)`: Проверяет, совпадают ли строки.
- `ck_assert_ptr_null(p)`: Проверяет, что указатель — `NULL`.
- `ck_assert(условие)`: Универсальная проверка (например, `ck_assert(a > b)`)
