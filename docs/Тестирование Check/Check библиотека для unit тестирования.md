https://linuxvox.com/blog/basic-unit-test-and-c-how-do-i-get-started/ - как начать модульное тестирование на С с использованием фреймворка Check. 

Подключается как 
```
#include <check.h>
```


Чтобы понять, как писать тесты с Check, нужно познакомиться с тремя его ключевыми понятиями
- **Test Case (Тестовый пример)**: Самая маленькая единица тестирования. Это конкретная функция, которая проверяет одно, специфическое поведение (например, `test_sensor_init_success`).
- **Test Suite (Набор тестов)**: Группа связанных тестов. Вы объединяете несколько Test Case в один Suite, чтобы логически сгруппировать их по модулям или функциональности.
- **SRunner (Тестовый раннер)**: Это "двигатель", который запускает все ваши тесты, собирает результаты и выводит отчет.

Тест оформляется с помощью макросов `START_TEST` и `END_TEST`
``` C
#include <check.h>
// Это тестовый пример (Test Case)
START_TEST (test_addition)
{
    int a = 5;
    int b = 5;
    int result = a + b;
    // Проверка (assertion): если 10 == 10, тест пройден
    ck_assert_int_eq(result, 10);
}
END_TEST
```
Макрос START_TEST создает функцию с именем, которую мы передали в скобках, в данном случае имя функции test_addition. 
	PS. Препроцессор С разворачивает Макрос в обычную функцию С (под капотом). 
Это имя вы будете использовать позже при добавлении теста в набор. См. далее

После того как тестовый пример создан нужно запустить его. 
``` C
Suite *s = suite_create("Container");
```
