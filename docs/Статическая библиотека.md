**Статическая библиотека в C** — это набор заранее скомпилированных объектных файлов (`.o` или `.obj`), собранных в один архивный файл. Она используется для **статической линковки**: на этапе компоновки компоновщик (linker) копирует прямо в ваш исполняемый файл те фрагменты библиотеки (функции, данные), которые реально используются.

1. **Скомпилировать в объектные файлы** (флаги `-c`):
    gcc -c mylib.c -o mylib.o
    gcc -c other.c -o other.o

2. **Упаковать в архив `.a`** (архиватор `ar`):
    ar rcs libmystatic.a mylib.o other.o
    - `r` — вставить файлы (заменить существующие)
    - `c` — не выводить сообщения о создании архива
    - `s` — создать индекс символов (для быстрой линковки)

3. **Использовать при линковке программы**:
    gcc main.c -L. -lmystatic -o program
    - `-L.` — искать библиотеки в текущей директории
    - `-lmystatic` — подключить `libmystatic.a` (префикс `lib` и расширение опускаются)

1. Определиться с набором функций, которые нужно реализовать
2.  Реализовать функции в отдельных .c файлах
	1. Каждая функция - в своём файле (это удобно для линковки: в итоговую программу попадут только используемые функции).
	2. Все функции должны быть объявлены в `s21_string.h`.
3. Создать makefile для автоматизации сборки(компиляции объектных файлов) и проверки тестов. 
4. Скомпилировать объектные файлы по 1 для каждого .c файла. Примерно следующим образом:
	gcc -c src/s21_strlen.c -o src/s21_strlen.o
	gcc -c src/s21_strcpy.c -o src/s21_strcpy.o
5. Сделать сборку статической библиотеки `s21_string.a`
